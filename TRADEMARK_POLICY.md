# OpenPort Trademark Policy

This policy applies to the `OpenPort` name, the OpenPort logo, and other
brand features used to identify official OpenPort releases and services.

Code in this repository is licensed separately under `AGPL-3.0-or-later`.
That software license does not grant trademark rights.

## Allowed without prior approval

- Truthful statements that your project uses, integrates with, or is based on
  OpenPort.
- Unmodified references to the OpenPort project in source code, attribution
  notices, issue reports, benchmarks, tutorials, and compatibility matrices.
- Internal use of the OpenPort name to evaluate, test, or deploy the software.

## Uses that require written permission

- Using the OpenPort name or logo in a way that suggests official
  endorsement, partnership, certification, or authorship.
- Offering a hosted or commercial service branded as `OpenPort` or
  `OpenPort Cloud`.
- Using the official logo, wordmark, or a confusingly similar brand in a
  product name, company name, package name, app listing, or domain name.
- Marketing a fork, derivative, or managed service as if it were the
  canonical OpenPort project.

## Forks and derivative services

If you modify this project and redistribute it, or offer it as a network
service, you must clearly distinguish your version from the official OpenPort
project. In practice that means:

- keep the original copyright and license notices
- avoid using the official logo by default
- rename the product if your fork changes behavior materially or is offered as
  a separate hosted service

## Commercial and partner use

For commercial materials, managed services, enterprise packaging, or other
partner-facing brand use, written approval is required.

For trademark requests, open a repository issue or contact the project
steward through official Accentrust channels.
