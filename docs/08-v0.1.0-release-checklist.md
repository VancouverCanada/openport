# v0.1.0 Release Checklist

Use this checklist before creating tag `v0.1.0`.

## 1. Environment

- [ ] Node 22 is active (`node -v` starts with `v22`)
- [ ] Clean working tree or only intentional release changes
- [ ] `origin` points to the public OpenMCP repository

## 2. Mandatory validation

Run in repository root:

```bash
npm ci
npm run gate
bash scripts/prepare-release.sh v0.1.0
```

- [ ] Build passes
- [ ] Test suite passes
- [ ] Local conformance profile passes
- [ ] Safety checks pass
- [ ] Boundary leak scan passes

## 3. Release assets

- [ ] `CHANGELOG.md` contains `0.1.0` entry
- [ ] `docs/releases/v0.1.0.md` is final
- [ ] `README.md` docs index is up to date
- [ ] License, SECURITY, CODEOWNERS are present

## 4. Git and tag

```bash
git add -A
git commit -m "chore(release): prepare v0.1.0"
git push origin main
git tag -a v0.1.0 -m "OpenMCP v0.1.0"
git push origin v0.1.0
```

- [ ] Annotated tag `v0.1.0` exists on remote

## 5. GitHub release

- [ ] Create GitHub Release from tag `v0.1.0`
- [ ] Title: `OpenMCP v0.1.0`
- [ ] Body copied from `docs/releases/v0.1.0.md`
- [ ] Mark as latest release

## 6. Post-release

- [ ] Verify README links and release notes render correctly
- [ ] Open milestone for `v0.1.x` patches
- [ ] Start `Unreleased` section in changelog for next cycle
