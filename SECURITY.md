# Security Policy

## Supported versions

Before `v1.0.0`, only the latest minor release is supported.

## Reporting a vulnerability

Please report security issues privately to the maintainers.

Include:

- impact summary
- affected endpoints or modules
- reproducible steps
- suggested mitigation (if available)

## Security principles

- least privilege defaults
- tenant boundary enforcement
- auditable action pipeline
- short-lived high-risk authorization windows
- fast revocation and kill switch
